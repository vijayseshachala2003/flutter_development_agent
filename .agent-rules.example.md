# Flutter Agent Rules

This is a Flutter application.

## Before editing
- Read `pubspec.yaml`.
- Inspect `lib/`.
- Identify state management.
- Identify routing.
- Identify API/service/model structure.
- Explain planned file changes before editing.

## Architecture rules
- Follow the existing folder structure.
- Do not create duplicate services, repositories, providers, models, or routes.
- Use the existing state management pattern.
- Prefer feature-first structure if already used.
- Keep widgets small and reusable.
- Avoid putting business logic inside UI widgets.
- Do not change backend API contracts unless asked.

## Flutter conventions
- Never use `BuildContext` across async gaps.
- Use `const` constructors where possible.
- Run `dart format .` after edits.
- Run `flutter analyze` after edits.
- Run tests when relevant.
